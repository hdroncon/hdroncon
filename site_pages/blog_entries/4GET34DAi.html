Efficient Applications and Solutions

Developing robust applications is not currently part of my day-to-day activities, and it's not a subject I'm well-versed in, so I can't go into great detail as the risk of making mistakes is high. However, it's an area I have a lot of interest in, and I try to absorb knowledge from articles I read whenever possible.

Even though I lack deep technical knowledge of the terminology, applications, and the true scope of the field, it's a fact that I enjoy applications, automation, problem-solving, and programming. Regardless of how simple and quick a task may be, if I understand that it's possible to automate that activity - whether to make the process even faster, standardize it, or simply reduce/eliminate the need for user interaction to prevent data entry errors - I believe it's worth dedicating time to automate it or, at the very least, create methods that facilitate user input.



Defining Paths

The development of graphical user interfaces (GUI) has been an area of interest for me for some time. Creating calculation routines and methods has been relatively straightforward and familiar tasks, but making the use of these resources user-friendly and independent of minimal knowledge of a particular software has always been a challenge. The initial steps were taken with Excel/VBA, which works reasonably well in this scope but has limitations due to the simple fact that it is not software designed for this purpose. Since I have some knowledge of Python, I explored the Tkinter library, which is intended for GUI development and serves its purpose well, but I was not entirely satisfied with its visual limitations. In this same vein, we have the excellent PyQT and PySide libraries, although I didn't venture into them at the time due to not fully understanding the legal means of distributing applications developed with them. Nowadays, these legal issues seem clearer to me, and I intend to use them in the near future.

In this project, I focused on the intersection of CSS, HTML, Javascript, and Python through the Eel library. While my knowledge of front-end technologies is quite limited, it's not (so) difficult to find resources on the internet for developing responsive websites. Moreover, the visual presentation that these tools offer is very appealing. Using Eel was a great choice to test (and improve) my skills in these web development tools, combined with Python, which I'm also quite comfortable with.



Budgeting App

In this application, I developed a user-friendly graphical interface for creating quotes - in this case, for vehicle sales. The process, which offers a high degree of customization in terms of option selection, becomes standardized and makes it easy to navigate through the stages of defining the package. Furthermore, the automatic generation of the quote itself eliminates the need for any calculations on the part of the estimator.

In the main column, you first need to choose the vehicle Model from the four available options.




The Accessories are also four - Steering Assistant, Braking Assistant, Automatic Transmission, and Hybrid Mode. These can be combined in any way, as the selection boxes are independent.




In order to implement more responsive features available in HTML, I created a second section for additional options. Here, you can choose the level of darkness for the window film, the radio model, and whether you want to test the vehicle on a racetrack (track day). For the radio, the default option is 'Padr√£o/Default', and to change it, you need to check the respective selection box. If you choose a different model from the default and then uncheck the box, the model automatically reverts to the default option.




Similar to the vehicle model, in the Packages section, you can choose only one of the services offered in the application.




Finally, Customer Information is required. There is an Observations field where the customer can write relevant notes about the request. Additionally, the Name, Email, and Phone Number are also requested so that the quote can be specifically directed to this customer, and they can be contacted in the future if necessary. After reviewing all the information, simply press the Generate Invoice button to create the document with the page's information.




The quotes are automatically saved in the \web\invoice directory with sequential nomenclature. At the top of the document, we have the header with general information and the document date.




Next, the items in the quote are itemized, along with the value associated with each of the accessories chosen through the application.




Finally, the observations entered during the quote creation are displayed below the budget value table.






Considerations

I'm still getting the hang of this blog's dynamics. In this post, I tried to explain the project in a general way, its purpose, and its application. In a parallel post, I delve into the technical aspects of what was done, the code, the logic, which will help me document this project and may serve as a source of information for my future work or for others as well. This post can be read here...

One of the main points of this application is that, among the inherent CRUD (Create, Read, Update, Delete) operations in databases, it only covers Create and Read. It's not possible to update or delete quotes. Since the database handling is a secondary factor in this project, I consider myself satisfied with this point.

Another focus here was on the automatic creation of a PDF file containing the database information. The library I decided to explore for this was FPDF, which has clear documentation and straightforward usage.

Some points for future improvements:

  1. add a field to display the percentage chosen for the darkness level of the window tint.
  2. limit the number of characters in the Notes, Name, Email, and Phone fields and make them mandatory.
  3. allow only numbers in the Phone field and check if the Email is valid (by checking if the field contains "@" or ends with ".com" or ".com.br," for example).
  4. a warning window if the quote was generated or if there was an error.
